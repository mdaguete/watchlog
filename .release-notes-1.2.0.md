# WatchLog 1.2.0

This release adds viewing-history import (Netflix), reworks the History/Stats
experience, unifies Search with Add, removes the Lists feature, and includes
several TMDB matching and stats accuracy fixes.

## Added

- **Viewing history import (Netflix)**: upload your Netflix `ViewingHistory.csv`
  to correct watched dates.
  - **Web flow** under user settings → *Import viewing history*: upload, then
    review a persisted batch of proposed date changes with per-row select and
    inline date editing; apply writes only the selected changes after an
    automatic database backup. Previous imports are listed and can be discarded.
  - **TMDB reconciliation**: series/movies not found in your library can be
    searched on TMDB and, on selection, added and marked watched with their
    Netflix dates (episode matching by title with chronological-order fallback).
    The unmatched list shows episode counts, season range, date range and
    sample titles to help identify the right match.
  - **CLI**: `watchdog netflix-dates <csv> [user-id] [--apply]` (dry-run by
    default; backs up the DB before writing).
- **Watched dates editing**: view and edit the watched date of any episode or
  movie; a new **editable monthly calendar**.
- **Shows filter by progress**: filter your shows by *to-watch* vs *caught up*.
- **`watchdog sync-stats [user-id]`**: recompute watch stats from the database
  (idempotent; handy after importing history).
- **Manual container build** workflow: build/push an image from any branch/tag,
  now also tagged immutably as `ghcr.io/mdaguete/watchlog:<ref>-<shortsha>`.

## Changed

- **History + Stats unified**: History now has two tabs — *Timeline* and
  *Stats*. Stats is the entry view; clicking a month in the activity heatmap
  opens that month's **calendar**. Calendar is no longer a separate tab.
- **Search + Add unified**: one search box now queries your **local library and
  TMDB** together. Local matches link to their pages; TMDB matches not already
  in your library get an **Add** button. The separate *Add* page redirects to
  *Search*.
- **Server administration** moved out of the settings page into a dropdown on
  the settings (⚙) menu, shown only to the admin user.
- **TMDB matching** on import/refresh now uses the authoritative **TheTVDB → TMDB**
  id mapping instead of name search, and movies are matched **year-aware**.
- **Watch stats** are now recomputed on every watch action (mark episode /
  season / movie watched, edit a watched date) and after imports, so the Stats
  page always reflects reality.
- **Removed the Lists feature** entirely (UI, API and database tables); it added
  little value.

## Fixed

- Duplicate shows created on import are de-duplicated (one show per title).
- TMDB refresh drops stale seasons/episodes from a previous wrong match.
- Movie watched dates are read correctly (movies page no longer appears empty).
- Search renders all results (a template error previously hid results).
- Stats: fixed a "NaN" year with no months caused by a malformed legacy period.
- Stats activity calendar: year labels no longer clipped on mobile.
- Fixed a database deadlock that hung the viewing-history import page.

## Notes

- **Migrations up to v18** run automatically on startup (a backup is written to
  `backups/` before migrating). v18 permanently drops the unused `lists` /
  `list_items` tables.
- When deploying the container, pull the new image explicitly (`--pull always`).
- TMDB features require a `TMDB_API_KEY`.
