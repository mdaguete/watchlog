# WatchLog 1.2.1

Adds streaming availability, a link to TMDB on detail pages, and a repair
command for shows affected by episode-numbering mismatches.

## Added

- **Streaming availability**: show and movie detail pages now display an
  **"Available on"** row of streaming-provider logos (TMDB watch providers).
  - The region is a **per-user** preference (set on the settings page, default
    ES), so each user sees providers for their own country.
  - Providers are cached per region and fetched on demand when opening a detail
    page (refreshed when older than 7 days).
- **Link to TMDB**: a "View on TMDB" link on show and movie detail pages.
- **`watchdog fill-aired <show-id> [user-id] [--apply]`**: marks a show's
  already-aired episodes as watched, dated by air date, to fix shows that
  appear partially unwatched due to season/episode numbering mismatches (e.g.
  TMDB "parts" vs TVTime). Never overwrites episodes already watched; dry-run by
  default and backs up the database before applying.

## Notes

- **Migrations up to v20** run automatically on startup (a backup is written to
  `backups/` before migrating): v20 adds a per-user region column and a
  per-region provider cache.
- To populate streaming providers for existing titles, open a show/movie detail
  page (fetched on demand) — no bulk action required. Set your region in
  Settings if you are not in Spain.
- When deploying the container, pull the new image explicitly (`--pull always`).
- Streaming providers and the TMDB link require a `TMDB_API_KEY`.
